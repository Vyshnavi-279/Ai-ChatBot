# spec.md — BVRIT College FAQ Chatbot (RAG + RAGAS)

This file is the single source of truth for the project. Keep it in the repo
root. Reference it in every Cline/Kiro prompt so the agent stays consistent
across sessions.

---

## 1. Project Overview

A Retrieval-Augmented Generation (RAG) chatbot that answers questions about
BVRIT college using a curated knowledge document (not the model's training
data). Every answer must cite the section and page it came from, and the
bot must refuse gracefully when the answer isn't in the document. The build
also ships an automated 8-dimension evaluation suite (including RAGAS
metrics) with a results dashboard.

## 2. Architecture

```
User Question
     │
     ▼
Streamlit Chat UI (st.chat_input)
     │
     ▼
Retriever (ChromaDB, top-k similarity search, optional section filter)
     │
     ▼
Grounding Prompt + Retrieved Chunks → LLM (generation)
     │
     ▼
Answer + Citations → Streamlit Chat UI (st.chat_message)
```

Offline / one-time step:
```
data/college_info.docx → Docx2txtLoader → RecursiveCharacterTextSplitter
   → text-embedding-3-small → ChromaDB (persisted to disk)
```

## 3. Tech Stack

| Component | Choice |
|---|---|
| Orchestration | LangChain |
| Document loader | `Docx2txtLoader` |
| Text splitter | `RecursiveCharacterTextSplitter` (chunk_size=800, overlap=120, split on headings first) |
| Embeddings | `text-embedding-3-small` via OpenRouter (1536 dims) — **same model for index + query** |
| Vector DB | ChromaDB, persisted locally to `./chroma_db` |
| Generation LLM | GPT-4o Mini via OpenRouter (swappable to Claude Sonnet / GPT-4o) |
| UI | Streamlit (`st.chat_input`, `st.chat_message`, sidebar) |
| Evaluation | RAGAS (faithfulness, answer relevancy, context precision, context recall) + custom 8-dimension LLM-as-judge suite |

## 5. Knowledge Base Document (Phase 0 — manual, done by you before coding)

`data/college_info.docx` must have 8 clearly-headed sections so the splitter
can chunk on structure:

1. About BVRIT — history, vision, mission, accreditations (NAAC, NBA)
2. Departments — B.Tech branches, specialisations, faculty count
3. Admissions — eligibility, entrance exams, process, key dates
4. Fee Structure — tuition by branch, hostel fees, scholarships
5. Placements — top recruiters, avg/highest package, placement %
6. Campus & Facilities — library, labs, hostel, sports, WiFi, transport
7. Faculty — key members, qualifications, research areas
8. Contact — address, phone, email, website, social media

Rules: factual only, no marketing copy, no invented facts.

## 6. Grounding Prompt Requirements

The generation prompt must enforce:
1. **Role** — BVRIT college information assistant
2. **Grounding rule** — answer ONLY from retrieved context, never training knowledge
3. **Citation format** — `[Section Name, Page N]` on every factual claim
4. **Refusal instruction** — graceful refusal + fallback contact when answer isn't in context
5. **Conflict handling** — if sections disagree, present both and flag the discrepancy

## 7. UI Design System — Light Theme (not dark, not stark white)

Use a warm, muted, professional light palette. Apply via `st.markdown` custom
CSS injected once at the top of `app.py`.

| Token | Hex | Use |
|---|---|---|
| `--bg` | `#F5F1EA` | app background (warm linen, not pure white) |
| `--sidebar-bg` | `#EDE7DC` | sidebar background |
| `--surface` | `#FFFFFF` | cards, chat bubbles |
| `--primary` | `#3D5A80` | headers, buttons, active states (muted navy-blue) |
| `--accent` | `#98C1D9` | badges, links, highlights (soft sky blue) |
| `--success` | `#6A994E` | passed tests, "in scope" badge |
| `--warning` | `#E09F3E` | warnings, medium scores |
| `--error` | `#BC4749` | failed tests, refusal badge |
| `--text` | `#293241` | body text (dark slate, not pure black) |
| `--border` | `#E4E0D8` | dividers, card borders |

Rules:
- No dark-mode toggle needed; ship light-only.
- Chat bubbles: user messages right-aligned in `--accent`-tinted bubble; bot messages left-aligned in `--surface` with `--border` outline.
- Citations render as small pill badges using `--primary` text on a light tint of `--accent`.
- Refusal responses get a `--error`-tinted badge labeled "REFUSED — not in knowledge base."
- Sidebar: knowledge base status, chunk count, retrieval settings (chunk size, overlap, top-k), optional section filter dropdown.
- Rounded corners (8–12px), soft shadows, generous padding — avoid a cramped, default-Streamlit look.

## 8. Eight Evaluation Dimensions

| # | Dimension | Test cases | Notes |
|---|---|---|---|
| 01 | Functional | 3 | format, citations present, completeness |
| 02 | Quality | 3 | factual match to doc, no hallucination |
| 03 | Safety | 2 | no outcome guarantees, no bias, no medical/legal/financial advice |
| 04 | Security | 2 | prompt injection resistance, no system-prompt leaks |
| 05 | Robustness | 3 | empty input, gibberish, very long input, emoji-only, mixed language |
| 06 | Performance | 2 | latency vs SLA (e.g. <10s) |
| 07 | Context | 2 | multi-turn follow-up resolution ("the first one") |
| 08 | RAGAS | 3 | faithfulness, answer relevancy, context precision, context recall |

Total: 20 test cases minimum.
